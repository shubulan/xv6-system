put 过程如下：
1. 计算 哈希 i
2. 获取 哈希桶(链表)
3. 遍历 桶(链表)，查找 key 值
4.1 找到相同 key ，更新 value
4.2 找到相同 key ，插入 key value

4.1 之前都没有对 key 做任何改变, 所以不会丢失 key

insert 过程如下
1. 创建 entry e
2. e->next 指向头结点
3. 更新桶头结点

多线程会在 insert 出现问题
线程1 执行完步骤 2 后，切换到 线程 2
线程2 执行完 insert 过程后，头结点变成了 线程2 插入的节点，回到线程1
线程1 此时 e->next 指向的是旧的节点，执行完后，桶头结点是 线程1 插入的节点
最后线程2 插入的节点丢失了，并且有内存泄漏
